Remarks

To send application arguments, such as file paths to open, use the ForwardFilesAsync(string, messageType, int, CancellationToken) method.

To receive application arguments, such as file paths to open, use the ApplicationForwarder<messageType> constructor. This will create a listener that will call the action when the message is received.

using (ApplicationForwarder<string[]> forwarder = new (message =>
{
    // Handle the files
}))

To send a message, use the ForwardFilesAsync(string, messageType, int, CancellationToken) method.

await ApplicationForwarder<string[]>.ForwardFilesAsync(pipeName, message);

The messages sent over the pipe are formattted as:

  • a 4-byte length prefix (int32)
  • the JSON payload (array of bytes) This allows the receiver to know how much data to read first, before reading the payload. This allows sending large payloads as well as payloads that contain control characters.

The sender and receiver can use a timeout to stop waiting for a response.

The sender can use a cancellation token to stop sending a message. This cancellation token is passed to the ForwardFilesAsync(string, messageType, int, CancellationToken) method.

The receiver uses a cancellation token to stop listening for incoming messages. This cancellation token is invoked when the Dispose() method is called.

Remarks

Typically, the sender will call this method, awaiting the result, and then exit the application.

The sender can use a cancellation token to stop sending a message. This cancellation token is passed to the ForwardFilesAsync(string, messageType, int, CancellationToken) method. If the cancellation token is signaled, the method will return false.
