# Glossary

This glossary defines terms and concepts used in the library documentation.

## A

**Assembly**
: A .NET library, such as DLLs or EXEs, containing executable code and resources.

**Attribute**
: In .NET, an attribute is a declarative tag applied to program elements such as classes, methods, properties, or parameters. Attributes provide metadata about the element, enabling custom behaviors and additional information.


## C

**Chart**
: A graphical representation of data that displays information visually.


## D

**Douglas-Peucker Algorithm**
: An iterative algorithm for reducing the number of points in a curve while preserving its overall shape.

## M

**Menu**
: A user interface element that organizes and presents a list of commands, actions, or options.

**Menu Item**
: An individual entry within a menu. Menu items represent executable actions or commands like "Open," "Save," or "Exit.".


## P

**Preference**
: A user- or application-specific setting that controls the behavior, appearance, or software configuration of software features. Preferences are stored persistently.

**Property**
: A member of a class or struct that provides access to data, typically encapsulating a getter and/or setter method for controlled reading and writing. Properties enable abstraction of internal state and implement logic for retrieving, validating, or updating values, distinguishing them from plain fields. In C#, properties are declared using the `get` and `set` accessors.


## R

**Redo**
: The operation that reapplies a change or action that was previously undone.


## S

**Single Instance**
: Refers to the pattern of allowing only one running copy of an application.


## U

**Undo**
: The operation that reverses or reverts a previous change or action, allowing the user or system to restore the prior state.
